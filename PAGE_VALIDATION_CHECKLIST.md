# 页面修改验证检查清单

> **用途:** 每个页面优化完成后，使用此清单进行验证
> **目标:** 确保修改后的页面质量和用户体验

---

## 📋 通用验证清单

每个页面修改完成后，必须完成以下检查：

### 1. 基础功能 (必须通过)

| 检查项 | 说明 | 状态 |
|--------|------|------|
| □ 页面正常显示 | 页面能够完整加载，无白屏或报错 | |
| □ 所有按钮可点击 | 按钮响应正常，点击后执行预期操作 | |
| □ 链接跳转正常 | 所有链接能正确跳转到目标页面 | |
| □ 表单可提交 | 表单验证正常，提交功能正常 | |
| □ 数据加载正常 | API 数据能正确获取和显示 | |

### 2. 视觉设计 (必须通过)

| 检查项 | 说明 | 状态 |
|--------|------|------|
| □ 颜色符合设计系统 | 使用 `--color-primary` 等变量定义的颜色 | |
| □ 字体统一 | 使用 `--font-primary` 字体族 | |
| □ 间距合理 | 使用 `--space-*` 变量定义的间距 | |
| □ 圆角一致 | 使用 `--radius-*` 变量定义的圆角 | |
| □ 阴影正确 | 使用 `--shadow-*` 变量定义的阴影 | |

### 3. 交互效果 (必须通过)

| 检查项 | 说明 | 状态 |
|--------|------|------|
| □ 悬停效果正常 | 鼠标悬停时有视觉反馈 | |
| □ 点击反馈正常 | 点击时有按下效果 | |
| □ 动画流畅 | 过渡动画平滑，无卡顿 | |
| □ 聚焦状态可见 | Tab 键导航时焦点清晰可见 | |
| □ 加载状态显示 | 数据加载时有 loading 提示 | |

### 4. 响应式设计 (必须通过)

| 检查项 | 说明 | 状态 |
|--------|------|------|
| □ 桌面端正常 (>1024px) | 大屏幕布局正确 | |
| □ 平板端正常 (641-1024px) | 中等屏幕布局自适应 | |
| □ 移动端正常 (<640px) | 小屏幕布局正确，无横向滚动 | |
| □ 触摸操作正常 | 移动端按钮大小足够，易于点击 | |

### 5. 技术检查 (必须通过)

| 检查项 | 说明 | 状态 |
|--------|------|------|
| □ 没有控制台错误 | 浏览器控制台无红色错误 | |
| □ 没有控制台警告 | 无重要警告信息 | |
| □ 没有 404 资源 | 所有图片/字体/样式正常加载 | |
| □ 性能正常 | 页面加载时间合理 | |

## ⚠️ 常见问题排查

### 问题1: 样式不生效

**可能原因:**
1. CSS 文件未正确导入
2. class 名称拼写错误
3. 样式被 Tailwind 覆盖

**解决方案:**
1. 检查 `main.jsx` 中的导入顺序
2. 使用浏览器开发者工具检查元素
3. 添加 `!important` 或调整选择器优先级

### 问题2: 悬停效果不工作

**可能原因:**
1. `:hover` 选择器被覆盖
2. 元素设置了 `pointer-events: none`
3. 父元素阻挡了鼠标事件

**解决方案:**
1. 检查元素的 CSS 层叠
2. 确保元素可接收鼠标事件
3. 使用更具体的选择器

### 问题3: 移动端布局错乱

**可能原因:**
1. 未使用响应式类
2. 固定宽度导致溢出
3. Grid/Flex 布局未适配

**解决方案:**
1. 使用 `.grid-cols-*` 响应式网格
2. 使用百分比或 `max-width`
3. 测试 640px 断点

### 问题4: 动画卡顿

**可能原因:**
1. 动画属性触发重排
2. 动画时间过长
3. 同时动画元素过多

**解决方案:**
1. 使用 `transform` 和 `opacity`
2. 使用 `var(--transition-fast)` 或 `var(--transition-normal)`
3. 减少同时动画的元素数量
